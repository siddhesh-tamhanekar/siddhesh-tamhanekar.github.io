Introduction

UniqueValidator (UValidator) is the open source validation library which is written in PHP with no external dependancy to validate the user input on the server side and keep your application safe without writing code for validation. You can specify validation rules as attribute in your HTML markup which can be used for server side validation. The library supports all HTML5 validation attributes.

Importance of Validations(For Beginners)

Never trust user input

Most of the basic working of dynamic web application or any interactive system is to take input from the user, process it and produce the output. While processing the input we should ensure that input is in valid format as the system defined. If input is in invalid format then the application can be vulnerable as well as it will be not usable e.g. if we are taking email without validating then we are not able to contact the person.

If the information is not in valid format, it's just a waste

Server Side Validations

  • Is server side validations are really important?
  • If I am validating the information at client side before submitting the form so validation takes place, Should I valid input twice?
  • Whether should I waste my server resources such as CPU time, memory on the validation where input is already validated?

The answer is with no doubt Yes. You should always do the server side validation before processing the data. The reason is client side validations can be easily compromised using various techniques. you can learn more about it here

Inspiration

Client side validations are essential for attractive, convinient User Experience and from the usability perspective on the other hand, for security reasons we need server side validation. Although there are tons of validation libraries which easing the works at both sides, writing the validation rules is most irritating task. Server side libraries are often force to write some piece of code for adding validation rules. and for client side we have to write some code in javascript too. The main purpose of library to avoid this duplication. The irritating and frustating jobs of writing validations on server side gives me inspiration for writing this UniqueValidator (UValidator).

Requirements

There is no special requirements to using UniqueValidator(UValidator).

  • PHP 5.x above
  • DOMDocument and DOMXPath Class(always Prebuilt in PHP)

Installation

The following steps you can follow to install/configure the UniqueValidator in your project.

  1. Unzip the downloaded zip file
  2. Put into your web application directory.
  3. To check everything is fine you can run demos located at 'demos' folder in UValidator
  4. browse to your_domain.com/Path_to_validator/demos/
  5. Use the library in your PHP Script by including it. 
    
<?php
 include "<your_application_directory>/UValidator/src/autoload.php";
 use UniqueValidator\UValidator;

 $uv = new UValidator("form_path#form_id");
 $result = $uv->valdate();
 if(!$result) {
 	# validation errors are occured you can get it by.
 	$errors = $uv->getErrors();
 }
?>
