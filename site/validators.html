Validators

Here is growing list of validators supported by Unique Validator (UValidtaor).

Required Validator

This validator validate if field is empty or not. It is also supported by HTML5 validations. You can use this validator in input as well as select tag.

You can use this by adding 'required' attribute in the input/select tag

Example: 

 <input required ... />

 <select required ..> ... </select>

Pattern Validator

This validator validate input against specified pattern. pattern should be specifed as PHP regular expression. for know more about pattern syntax of php regular expression refer here

You can use this by adding attributes ' pattern="add pattern here" ' in the input tag

Example: 

 <input type="text" pattern="[a-zA-Z]*" />

Text Validator

This validator validates if field contains the input is as specified for doing it it's depended on other attributes like min,max

Validator is automatically added when markup specify input tag type attribute as text.

Supported Attributes:
  • min : for minimum length of string supported by field.
  • max : maximum length of the string supported by field.
Example: 

 <input type="text" min="3" max= "20">

Number Validator

This validator validates if field contains the numeric value and it's in the defined range.

Validator is automatically added when markup specify input tag type attribute as number.

Supported Attributes:
  • min : for minimum number supported by field.
  • max : maximum number supported by field.
Example: 

 <input type="number" min="3" max= "20">

Date Validator

This validator validates date, it's format and it's in provided range.

Validator is automatically added when markup specify input tag type attribute as date.

Supported Attributes:
  • min : for minimum date supported by field.
  • max : for maximum date supported by field.
  • format : format of the date as per PHP date formats. Default format is (Y-m-d)
Example: 

 <input type='date' min="2010-12-12" max="2016-12-12" data-format="Y-m-d" />

Url Validator

This validator validates url against regular expression .

Validator is automatically added when markup specify input tag type attribute as url.

Example: 

 <input type="url" >

Checkbox Validator

Validator validates the submitted value is valid (one of the checkboxes value) and it also checks minimum and maximum checked checkboxes properly.

Validator is automatically added when markup specify input tag type attribute as "checkbox".

Supported Attributes:
  • min : for minimum required checkboxes.
  • max : maximum checkboxes can be checked.
If you are using multiple checkboxes with same name then place all the validation attributes at first checkbox
Example: 

 <input type="checkbox" name="fav_sports[]" value="cricket" min=1  > Cricket
 <input type="checkbox" name="fav_sports[]" value="football" > Football

Radio Validator

Validator validates the submitted value is valid (one of the radio input value value)

Validator is automatically added when markup specify input tag type attribute as "radio".

If you are using multiple radio inputs with same name then place all the validation attributes at first radio input field
Example: 

 <input type='radio' name="gender" value="male" required> Male
 <input type='radio' name="gender" value="female" > Female

Select Validator

Validator validates the submitted value is valid (one of the option value) and it also checks minimum and maximum checked properly.

Validator is automatically added when markup specify select tag.

Supported Attributes:
  • min : for minimum options should be selected if multiple attribute is added.
  • max : maximum options can be selected if multiple attribute is added.
  • data-checkvalue : if this attribute is present validator will not parse all the options for listing valid values
    The attribute is useful when your dropdown has large no of values like country, city, state dropdown has if we maintain all values at server side we require large memory which doesn't seem to be a feasible solution so we recommend if you have large values for select then use this attribute and use custom callback function to validate valid value.
Example: 

 <select name='select'> ....</select>

 <select multiple min="2" max="5" name='select'> ....</select>

Textarea Validator

same as Text Validator

Readonly Validator

This validator ensures that the value of field will never changed by user

Validator is automatically added when markup specify input tag has readonly

Example: 

 <input type="hidden" name="user_id" value="123" readonly />

Default Validator

This validator ensures that the value of field should changed by user ie form cannot be submitted before changing the input field value.

it is not html5 validator it's custom validator given by Uniquevalidator so you have to use it by prefixing 'data-uv-'

Validator can be added by adding data-uv-default

Example: 

 <input type="text" name="full name" value="Siddhesh Tamhanekar" readonly />
